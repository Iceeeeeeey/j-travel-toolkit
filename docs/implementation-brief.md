# J人旅行神器实施文档

## 核心定位

J人旅行神器是一个给“爱做计划，但出门后不想翻密密麻麻文档”的人用的旅行计划 app 生成 skill。

用户不需要理解 PWA、静态站点、GitHub Pages 或部署流程。用户只需要：

1. 说一句：`使用 J人旅行神器，帮我生成旅行计划 app`
2. 拿到一份固定 Excel 模板
3. 在一张表里填核心行程
4. 把填好的 Excel 发回给 AI
5. 收到一个手机浏览器能打开、能添加到主屏幕的公开链接

## 用户输入边界

用户只填写一个 sheet：`行程`。

模板只有一张表，字段为：

- `日期`
- `星期`
- `城市`
- `时间`
- `行程`
- `交通`
- `住宿`

模板里保留约 3 天示意数据，并在第一行提醒用户：上传前删掉示意数据，再填自己的真实行程。

不要要求用户自己设计 Excel 格式。用户上传非模板结构时，不要猜，先给模板并让用户把内容迁移进去。

## AI 补全职责

用户只填核心行程，AI/skill 负责补充旅行 app 里常用的信息。

生成前应根据城市、行程、交通、住宿补：

- 城市级出门提醒
- 隐私提醒
- 交通提醒
- 购票和景点搜索入口
- Klook 搜索入口
- Google Maps 搜索入口
- 天气入口
- 汇率入口

规则：

- 能确定的链接用确定链接。
- 不能确定具体商品页时，用搜索页，不要假装已经找到了准确商品。
- 不内置地图组件，只放外部链接。
- 不把护照号、订单号、详细房号、手机号等敏感内容发布到公网。

## App 信息架构

保留 Lovable 版本的核心结构和交互：

- 顶部旅行标题和天数
- 横向日期选择器
- 今日行动卡
- 时间线行程
- 住宿卡片
- 底部导航
- 页面：`行程`、`贴士`、`交通`、`快捷`
- 物品清单支持本地勾选

## 视觉方向

尽量接近用户 Lovable 欧洲旅行版本：

- 移动端优先
- 暖色旅行感
- 紫色、玫红、橙色的高识别主视觉
- 卡片轻盈、有层次
- 横向日期选择器明确
- 行程时间线清楚

优化要讲 ROI：不重写为复杂设计系统，不引入高维护成本，只在模板 CSS 和结构上提升观感。

## 发布闭环

Skill 的交付物不是本地 `127.0.0.1`。

完成标准是返回公网链接：

```text
https://<github-user>.github.io/<trip-repo>/
```

默认策略：

- 一次旅行一个公开 GitHub 仓库
- 源码推到 `main`
- 静态站点推到 `gh-pages`
- 用户手机打开 GitHub Pages 链接后添加到主屏幕

安装/使用说明必须提醒用户：

- 需要有 GitHub 账号
- 需要让本机 git 能 push 到 GitHub
- 可用 SSH key 或 GitHub CLI
- GitHub Pages 是公开链接，不要放敏感信息

## 真实验收标准

不能只跑本地预览。

验收必须从 0 开始：

1. 卸载已安装 skill
2. 从 GitHub 安装 skill
3. 模拟用户口语化请求
4. Skill 给模板
5. 用填好的模板生成 app
6. 创建/配置 GitHub 公开仓库
7. 发布到 GitHub Pages
8. 打开公网链接验证页面可用
9. 验证手机宽度无明显溢出
10. README 包含模板下载、安装、使用、GitHub 凭据说明和截图
